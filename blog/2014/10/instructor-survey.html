In September, Greg Wilson wrote a blog post on "Building better teachers" that was inspired by an issue brought up by John Blischak on reporting time spent on lessons. In order for us capture this information, we had to revamp the post workshop instructor surveys.

First round of data collection

We began by asking all the instructors who participated in teaching this past summer to report time spent teaching various subjects (Shell, Python, Git, etc) and anything that worked or did not work during the workshop. A pull request provided the first pieces of data for instructors with regards to time spent on each topic.

Git Python Shell SQL
n 8 5 8 1
min (hours) 1 4 1.5 3
median (hours) 2.25 4.5 3 3
mean (hours) 2.06 4.9 2.8 3
max (hours) 3.5 6 3.5 3
Other than Git, the times were fairly consistent with one another. I say "other than git", because Damien manages to teach Git in 1 hour by not having students type along. Rather he has them watch him go through the git commands. There is a recording of a workshop he ran for us to listen to. Additionally, we only had 1 workshop that responded that covered all 4 topics. SQL is usually left out in place of Python.

Revamped survey

The individual questions for the survey are at the end of this post. However, there are a few questions for the other instructors to keep in mind while reviewing the survey.
  • What order should the questions be presented in? In questions by lesson, or lessons by questions. That is, should we ask all the questions pertaining to Shell, then all the questions relating to Python, etc... (questions by lesson) or should we have a section for time, deviation from the repo material, etc and have all the topics under each section (lessons by question)
  • Is there any other information you think would be useful to track? I have added swc installer and ipython notebook use/installation/problems
  • Would you take this survey? Is it too long? the question format is bad, etc. Any feedback on the survey design and layout is greatly appreciated.
  • Are the reference examples for the likert scale questions well spaced? The reason for the reference example is to provide some context for instructors so we can have some standardized set of responses.
  • How feasible is it to track individual topics in a lesson? John mentioned that time spent on an entire lesson (i.e., total time for python) is not that useful when instructors are trying to pace the lesson. For example, if there is only 30 minutes left for Python, is there enough time to teach the v5 testing material?

The live survey can be found here The static post of the survey is replicated below:

  2. Last name (text box - required)

  5. Did you use the SWC Installer? (radio button - required): Yes, No, I was not aware there was one, Other (text box)
  6. Any comments about the SWC installer? (text box - optional)
  7. What topics did you teach and how long did it take (in hours)? (matrix of radio buttons - required) On the Y-axis are the lessons (Shell, Git, Python, etc...) the X-axis has time in hours (1, 1.5, 2, ..., 7.5, NA)
  8. If you taught 'other' please describe what you taught and how long it took (e.g., "SVN (3.5); SPSS (3); SAS(2.5)")
    • How much of the SWC material did you use for Shell?
    • How much of the SWC material did you use for Git?
    • How much of the SWC material did you use for Python?
