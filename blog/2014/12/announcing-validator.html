Thanks the awesome work of Andy Boughton we now have a validator to check if our lessons, more or less, follow the template we proposed. Continue reading to know what the validator does, the improves that we still need to do and how you can help.

Lesson Template Specification

During last month, Greg Wilson ported to the new template some of Software Carpentry's lessons as he announced a few days ago. From this work small changes was made at the specification that is already cover by the validator but still need to be sumarize. For exemple, at topic files authors should use 

> ## Learning Objectives
>
> * Learning objective 1
> * Learning objective 2
instead of 
> ## Learning Objectives {.objectives}
>
> * Learning objective 1
> * Learning objective 2

Current State

The validator uses Python CommonMark parser to create a abstract syntax tree for every Markdown file at pages directory that our script uses to validate the files.

Andy choosed to use a Object Oriented approach for the script that add a little of complexity to it but makes it more reusable. For every type of file the validator has a class that is a children of MarkdownValidator so if you need to change how the validator works for pages/LICENSE.md you just need to look at the classes LicensePageValidator and MarkdownValidator.

The validator checks for:

  • valid YAML header,
  • correct heading level,
  • correct block structures (e.g. prerequisites),
  • required sections,
  • valid links,
  • valid glossary entries.

What you can do

  • Summarize the changes at the specifications.
  • Test the validator with 
    $ tools/check pages
  • Report suggestions and bugs at GitHub
  • Send pull request to issues related to it.
