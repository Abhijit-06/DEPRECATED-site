Software Carpentry is an open project. Like all such projects, it needs your help to grow. If you see a mistake in on our website, please add a comment to the appropriate page or mail us and we'll fix it as quickly as we can.

Fork Us on GitHub

Pull requests are always welcome, or you can mail us to talk about what you'd like to do and how we can help.

New Lessons and Examples

We welcome new lessons, particularly extended examples that show how the things we teach fit together. For an example, take a look at the invasion percolation example, which covers topics as diverse as restructuring code to make it more testable, performance profiling, and the advantages of lazy data structures. Such examples can be specific to different research disciplines, which often makes them the most convincing part of our bootcamp.

Related Projects

We teach using many people's tools, and contributions to those will always help us. In particular:

  • ipythonblocks and skimage.novice have let us adopt a media-first approach to teaching programming. Enhancements to either of these are very welcome.
  • BrowserCast will, when finished, allow us to replay an IPython Notebook in sync with a voiceover—in other words, to use the Notebook for screencasting without the large file sizes and loss of first-class content that plague current screencasting options.
  • There are lots of plugins for the IPython Notebook that would make our job easier. We'd particularly like something like Regex Pal for teaching regular expressions and an embedded vector drawing widget for doing diagrams. If you're interested in working with a great bunch of developers on a tool that's changing the way scientists work every day, please get in touch.
